## #3 ‚Äî in-process crash containment gaps (Medium‚ÄìHigh)

`src/asmtest.c`:
- **Alternate signal stack.** `install_handlers` now installs a 64 KiB
  `sigaltstack` and sets `SA_ONSTACK` on all four crash handlers (SIGSEGV/BUS/ILL/FPE)
  and SIGALRM, so a **stack-overflow SIGSEGV** in a routine under test can still
  run the handler on the in-process paths (`--no-fork`, `--bench`, fork/pipe
  fallback) instead of killing the whole runner. (A fixed 64 KiB because `SIGSTKSZ`
  is a runtime value on modern glibc, unusable as an array size.)
- **Stale `jmp_buf` alarm race.** `run_one` now cancels the per-test timeout
  (`alarm(0)`) immediately after the body, **before** the teardown `sigsetjmp`
  frame ‚Äî so a late SIGALRM can no longer fire after that frame returns but before
  the cancel, `siglongjmp`-ing into a stack frame that no longer exists (UB).
- The async-signal-unsafe SIGALRM handler (`snprintf` + `siglongjmp` into
  possibly-interrupted `malloc`/`printf`) is inherent to the in-process longjmp
  model; the `snprintf` writes a fixed buffer with no allocation, and the forked
  model (default) avoids the interrupted-lock hazard entirely. Left as a documented
  residual.

## #8 ‚Äî shell robustness in the clean-room trust anchor (Medium)

- `scripts/assert-clean-path.sh` now runs under `set -u`; the temp-path escape
  hatch uses `${TMPDIR:-/nonexistent}` so an **unset `$TMPDIR` can no longer
  degrade to the glob `*`** and accept a leaked library as "clean". Optional env
  vars are `${VAR:-}`-guarded for `set -u`, and an **unresolvable install prefix**
  (empty `$pp` ‚Üí `/*`) is now a hard error rather than an accept-anything glob.
  *Verified:* a `build/` path and a leaked path outside a real prefix (with
  `$TMPDIR` unset) are both rejected; a `/tmp` extraction and an under-prefix path
  are accepted.
- `scripts/clean-room-test.sh` ‚Äî the summary print uses `printf '%b' "$summary"`
  so a `%` in a resolved path can't be treated as a format directive.

## #9 ‚Äî Lua emu binding drops integer args (Medium)

`bindings/lua/asmtest.lua` ‚Äî `Emu:call_fp` and `Emu:call_vec` no longer hardcode
`nil, 0` for the C `iargs`/`niargs` parameters; they accept `opts.iargs` and
marshal them into the GP argument registers, so a Lua caller can test a mixed
routine like `f(long n, double x)`. Matches the peer bindings (Python/Node/Ruby/Go)
and the C support in `src/emu.c`. `luajit -bl` clean.

## #10 ‚Äî emulator doesn't reset state between calls on a reused handle (Medium)

`src/emu.c` ‚Äî new `emu_x86_zero_regs()` / `emu_arm64_zero_regs()` helpers zero the
GP + vector register file (and reset RFLAGS / NZCV) at the start of each call
(SysV, Win64, and AAPCS64 setups), so a routine that reads a register or vector
lane the caller did not set gets a deterministic 0 rather than the previous call's
state ‚Äî removing the reused-handle nondeterminism footgun (all bindings, and the
fuzzer/mutation harness, hold one long-lived handle). Guest **mapped** memory is
intentionally preserved (the `emu_map`/`emu_write` preload feature); the internal
stack is re-based per call.

*Verified:* on a reused x86 handle, a routine returning `rbx` after a prior call
left `rbx = 0x1234` now reads `0x0`. `make emu-test` unchanged on both x86-64 (45/47,
the 2 failures pre-existing flaky fuzz tests) and linux/arm64 (33 pass + 12 skip,
same 2 pre-existing) ‚Äî confirming the AArch64 register-zeroing clobbers no arg
registers.

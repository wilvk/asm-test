# bindings-parity-allow.txt — intentional exemptions for check-bindings-parity.
#
# The parity gate (scripts/check-bindings-parity.sh) requires every language
# binding to wrap every symbol declared in the native-trace tier headers (the
# TIER_HEADERS default: asmtest_hwtrace.h, asmtest_drtrace.h, asmtest_trace_auto.h,
# asmtest_ptrace.h, asmtest_codeimage.h) — the surface that is uniformly mirrored
# across all ten bindings and the one most often extended "across all bindings"
# in a single change. List here the (binding, symbol) pairs that are deliberately
# NOT wrapped.
#
# Syntax:  <lang|ALL> <symbol>          # everything after '#' is a comment
#   ALL   exempts the symbol for every binding.
#   <lang>  exempts it for one binding, where <lang> is one of:
#           python cpp rust zig node java dotnet ruby lua go
# Blank lines and lines beginning with '#' are ignored.
#
# Stale entries fail the gate (the binding actually wraps the symbol now, or the
# symbol left the tier headers), so this file cannot rot silently — remove an
# entry once it is no longer needed.

# Optional Keystone-only convenience helper (assemble + materialize in one call);
# every binding wraps the lower-level asmtest_exec_alloc instead, so none bind it.

# GC-safe capturing upcall. It still wraps every other asmtest_descent_* symbol.

# Scoped-tracing additive C-layer primitives (scoped-tracing-core-plan; summary in
# docs/scoped-tracing-implementation.md). The bindings' scope construct ships against
# the NAME-keyed path — asmtest_hwtrace_try_begin / _render, wrapped in all ten — as the
# header's §1 note states. The symbols below are the additive, host-tested C surface the
# bindings deliberately do NOT wrap; they carry no per-binding caller today. Wire one in
# (and drop its line) when a binding grows a use for it.
ALL asmtest_hwtrace_begin_scope       # §1 handle-keyed begin: additive over the name-keyed try_begin path the shims use (its render sibling, asmtest_hwtrace_render_scope, is now wrapped in all ten via call_scoped)
ALL asmtest_hwtrace_render_versioned  # §1/§D4 version-aware render: needs a code-image handle no shim carries
ALL asmtest_hwtrace_stitch            # §D4 async-hop merge core: pure C concatenation, host-tested (test_stitch_slices)
ALL asmtest_hwtrace_stealth_trace     # §D3 out-of-process ptrace-stealth stepper: dotnet (AsmTrace.Method outOfProcess), node + java (HwTrace.stealthTrace) wrap it as the crash-proof managed-capture route; the other seven reach it C-internally / gain it when they grow a stealth scope
ALL asmtest_hwtrace_region_name       # §3.1(c) address→name reverse resolver: whole-window noise attribution, C-level
ALL asmtest_hwtrace_symbolize_bucket  # §3.1(c) IP bucketer: whole-window noise attribution (dotnet wraps it as HwTrace.SymbolizeBuckets for its runtimebuckets example; the other nine gain it when they grow a whole-window scope), host-tested
ALL asmtest_hwtrace_begin_window      # §Z1 region-free whole-window trio: the .NET managed tier's AsmTrace substrate
ALL asmtest_hwtrace_end_window        #   (dotnet wraps it); the other bindings ship the name-keyed try_begin path and
ALL asmtest_hwtrace_render_window     #   gain the trio when they grow a scope construct — same posture as §1 above
ALL asmtest_hwtrace_attribute_window  # §3.1(c) whole-window noise attribution: same family as region_name/symbolize_bucket
ALL asmtest_hwtrace_call_scoped       # Option B lazy-arm NAMED-call scope (managed-singlestep-lazy-arm-plan): dotnet wraps it (AsmTrace.Method's Invoke); the other nine ship the registry-FREE call_scoped_ex form instead (no fixed-table slot), so none bind the registry-keyed call_scoped
ALL asmtest_hwtrace_call_scoped_fp    # FP (double…)->double sibling of call_scoped: dotnet wraps it (AsmTrace.Method's Invoke FP path); the integer call_scoped_ex now ships in all ten, but the FP sibling stays dotnet-only until a binding traces a double->double leaf
ALL asmtest_hwtrace_stitch_handles    # §D0.4 live-producer bridge for asmtest_hwtrace_stitch: dotnet wraps it (AsmStitchedTrace, the async-hop producer); the other nine consume the shipped stitch core when they grow a stitched-scope form — same posture as asmtest_hwtrace_stitch above, host-tested (test_stitch_handles)
ALL asmtest_ptrace_trace_attached_windowed  # §D3 whole-window multi-region capture: driven by the bundled asmtest-stealth-helper binary + C examples, not per-binding FFI
ALL asmtest_amd_snapshot_trace        # AMD deterministic boundary LBR snapshot: C-level callback capture, reached through the region begin/end markers (snapshot opt-in), not direct FFI
ALL asmtest_amd_msr_trace             # AMD MSR-direct LBR snapshot (src/msr_lbr.c): C-level callback capture reading /dev/cpu/N/msr, self-hosted CAP_SYS_ADMIN lane only — same posture as asmtest_amd_snapshot_trace
ALL asmtest_amd_msr_available         #   substrate probe for the above (amd_lbr_v2 + /dev/cpu/N/msr); C-level gate, not per-binding FFI

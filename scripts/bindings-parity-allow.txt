# bindings-parity-allow.txt — intentional exemptions for check-bindings-parity.
#
# The parity gate (scripts/check-bindings-parity.sh) requires every language
# binding to wrap every symbol declared in the native-trace tier headers
# (asmtest_hwtrace.h, asmtest_drtrace.h) — the surface that is uniformly mirrored
# across all ten bindings and the one most often extended "across all bindings"
# in a single change. List here the (binding, symbol) pairs that are deliberately
# NOT wrapped.
#
# Syntax:  <lang|ALL> <symbol>          # everything after '#' is a comment
#   ALL   exempts the symbol for every binding.
#   <lang>  exempts it for one binding, where <lang> is one of:
#           python cpp rust zig node java dotnet ruby lua go
# Blank lines and lines beginning with '#' are ignored.
#
# Stale entries fail the gate (the binding actually wraps the symbol now, or the
# symbol left the tier headers), so this file cannot rot silently — remove an
# entry once it is no longer needed.

# Optional Keystone-only convenience helper (assemble + materialize in one call);
# every binding wraps the lower-level asmtest_exec_alloc instead, so none bind it.
ALL asmtest_asm_exec_native

# Call descent (asmtest_descent_t) upcall setters. The Rust binding is allow-set
# only: descent targets are selected via allow_region/deny_region, never a
# capturing-closure resolver, because a Rust extern "C" fn pointer cannot host a
# GC-safe capturing upcall. It still wraps every other asmtest_descent_* symbol.
rust asmtest_descent_set_resolver   # allow-set only: no capturing-closure upcall
rust asmtest_descent_set_denylist   # allow-set only: no capturing-closure upcall
cpp asmtest_descent_set_resolver   # allow-set only: dlopen FFI hosts no capturing upcall
cpp asmtest_descent_set_denylist   # allow-set only: dlopen FFI hosts no capturing upcall
zig asmtest_descent_set_resolver   # allow-set only: std.DynLib hosts no capturing upcall
zig asmtest_descent_set_denylist   # allow-set only: std.DynLib hosts no capturing upcall
ruby asmtest_descent_set_resolver   # allow-set only: Fiddle hosts no GC-safe capturing upcall
ruby asmtest_descent_set_denylist   # allow-set only: Fiddle hosts no GC-safe capturing upcall

# Scoped-tracing additive C-layer primitives (scoped-tracing-core-plan; summary in
# docs/scoped-tracing-implementation.md). The bindings' scope construct ships against
# the NAME-keyed path — asmtest_hwtrace_try_begin / _render, wrapped in all ten — as the
# header's §1 note states. The symbols below are the additive, host-tested C surface the
# bindings deliberately do NOT wrap; they carry no per-binding caller today. Wire one in
# (and drop its line) when a binding grows a use for it.
ALL asmtest_hwtrace_begin_scope       # §1 handle-keyed scope trio: additive over the name-keyed path the shims use
ALL asmtest_hwtrace_render_scope      # §1 handle-keyed render: additive over asmtest_hwtrace_render (name-keyed)
ALL asmtest_hwtrace_render_versioned  # §1/§D4 version-aware render: needs a code-image handle no shim carries
ALL asmtest_hwtrace_stitch            # §D4 async-hop merge core: pure C concatenation, host-tested (test_stitch_slices)
ALL asmtest_hwtrace_stealth_trace     # §D3 out-of-process ptrace-stealth stepper: C-internal to the scope façade
ALL asmtest_hwtrace_region_name       # §3.1(c) address→name reverse resolver: whole-window noise attribution, C-level
ALL asmtest_hwtrace_symbolize_bucket  # §3.1(c) IP bucketer: whole-window noise attribution, host-tested

# third-party-digests.txt — trusted integrity anchors for the third-party engines
# this project downloads or builds and then bundles into user-facing packages
# (repo-review B5). Without these, integrity rested only on a version string in a
# URL or a *mutable* git tag: a moved tag or a swapped release asset would ship an
# altered engine under this project's name with every release job still green.
#
# The fetch/build scripts (fetch-dynamorio.sh, build-keystone.sh, build-capstone.sh)
# verify what they pull against the matching line here and FAIL on any mismatch or
# on a missing entry — so a version bump that forgets this file cannot ship unpinned.
# Regenerate after a bump with:  scripts/refresh-thirdparty-digests.sh
#
# Format (whitespace-separated):  <kind>  <name>  <version>  <algo>:<value>
#   tarball-sha256 — SHA-256 of the downloaded release archive (as GitHub records it)
#   git-commit     — the immutable commit a version tag must still resolve to
